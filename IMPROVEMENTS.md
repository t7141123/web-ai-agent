# 🧬 GOLEM Agent 自我進化機制改善報告

## 📋 改善概述

本次改善計畫的目標是讓 GOLEM Agent 成為一個**真正能夠自我進化的智能體**，通過 Web Gemini 進行自主學習和成長。

### 新增核心模組

| 模組 | 檔案 | 功能說明 |
|------|------|----------|
| **SelfEvolutionCore** | `core/self-evolution.js` | 真實自我進化核心，追蹤決策、發現模式、執行實驗 |
| **SelfMonitor** | `core/self-monitor.js` | 自我監控與報告系統，健康分數、停滯偵測、突破記錄 |

### 改善現有模組

| 模組 | 改善內容 |
|------|----------|
| **Brain** (`core/brain.js`) | 整合自我進化核心和監控系統，增強決策追蹤 |
| **Agent** (`core/agent.js`) | 新增監控和實驗指令，完善 shutdown 機制 |
| **PromptEvolver** (`core/prompt-evolver.js`) | 增加針對性優化，分析負面回饋模式 |
| **GeminiWebClient** (`browser/gemini-web.js`) | 增強抗阻擋能力，智能恢復機制 |

## 🆕 新增功能詳解

### 1. SelfEvolutionCore - 真實自我進化核心

#### 核心哲學
```
真正的自我進化 = 觀察自己的行為 → 發現模式 → 主動修正 → 驗證改進
```

#### 三大支柱

**1. 行為追蹤 (Behavior Tracking)**
- 追蹤每種決策類型：
  - `ROUTE_CHOICE`: 選擇 API 還是 Web
  - `SKILL_APPLICATION`: 選擇使用哪個技能
  - `ACTION_SELECTION`: 選擇執行哪個動作
  - `CONFIDENCE_CALIBRATION`: 信心度評估
  - `FAILURE_RECOVERY`: 失敗後如何恢復

- 記錄決策上下文和結果
- 每 50 個決策自動觸發深度模式分析

**2. 模式挖掘 (Pattern Mining)**
- 自動聚类失敗模式（≥3 次重複失敗）
- 自動聚类成功模式（≥3 次高信心成功）
- 深度分析根本原因
- 提煉可重複使用的策略

**3. 主動實驗 (Active Experimentation)**
- 根據失敗模式自動生成實驗
- 實驗類型：
  - `NEW_STRATEGY`: 嘗試新策略
  - `SKILL_ADJUSTMENT`: 調整現有技能
  - `THRESHOLD_CHANGE`: 調整閾值
  - `PROMPT_VARIATION`: 嘗試不同 prompt 變體
- 累積 5 次試驗後自動得出結論
- 成功實驗自動應用到系統中

#### 使用指令
```bash
insights     # 查看自我進化洞察
experiments  # 查看活躍實驗和結果
```

---

### 2. SelfMonitor - 自我監控與報告系統

#### 核心功能

**1. 即時監控**
- 總互動次數追蹤
- 每日統計（成功/失敗、信心度、路由選擇）
- 技能使用頻率
- 運行時間統計

**2. 停滯偵測**
- 觸發條件：
  - 成功率 < 40%
  - 平均信心度 < 50%
  - 需要至少 5 次互動
- 自動分析可能原因
- 生成改進建議
- **自動觸發進化週期**

**3. 突破記錄**
- 記錄高信心度（>0.9）的成功互動
- 保留使用的技能和動作類型
- 最多保留 50 個突破

**4. 報告生成**
- 每 10 次互動檢查是否生成報告
- 每日報告自動生成
- 健康分數計算（0-100 分）
  - 成功率加成（+20）
  - 信心度加成（+15）
  - 進化週期加成（+10）
  - 停滯扣分（-10/次）

#### 健康分數說明

| 分數 | 狀態 | 顏色 |
|------|------|------|
| 70-100 | 優秀 | 🟢 綠色 |
| 40-69  | 良好 | 🟡 黃色 |
| 0-39   | 需改進 | 🔴 紅色 |

#### 使用指令
```bash
monitor  # 查看系統監控狀態和健康分數
report   # 生成並查看進化報告
```

### 3. Prompt 針對性優化

#### 改善內容

**舊版問題**
- 只根據整體滿意度觸發優化
- 沒有分析具體失敗模式
- 優化方向不夠精確

**新版改進**
1. **模式分析**
   - 分析重複出現的負面回饋（≥2 次）
   - 識別失敗信號類型（語言模式、重複主題等）

2. **針對性優化**
   - 根據具體失敗模式提出改進方案
   - 明確指出要修改的 prompt 部分
   - 說明預期改進效果

3. **自動應用**
   - 優化後自動保存新版本
   - 記錄版本變更原因

---

### 4. 瀏覽器抗阻擋能力增強

#### 阻擋偵測增強

**新增偵測信號**
- URL 模式：
  - `unsupported-browser`
  - `continue-unsupported`
- DOM 選擇器：
  - `[class*="blocked"]`
  - `[class*="captcha"]`
  - `button:has-text("Sign in")`
  - `button:has-text("登入")`
- 頁面內容：
  - "Unsupported browser"
  - "Please sign in"
  - "驗證碼"
  - "登入"
  - "無法載入"

**頁面狀態檢查**
- 檢測頁面是否完全空白
- 檢測頁面是否只有錯誤訊息

#### 智能恢復機制

**策略 1：訪客模式恢復**
- 偵測到登入提示時
- 自動點擊「繼續作為訪客」按鈕

**策略 2：配額限制恢復**
- 偵測到 quota/limit 錯誤
- 自動切換新分頁

**策略 3：錯誤頁面恢復**
- 偵測到錯誤頁面
- 重新導航到 Gemini 首頁

**預設策略**
- 換新分頁重試

## 🎯 自我進化流程圖

```
用戶互動
   │
   ▼
Brain.think() ──────────────→ 追蹤路由決策
   │                              (selfEvolution)
   ▼
Executor.execute() ─────────→ 追蹤技能應用
   │                              (selfEvolution)
   ▼
結果回饋 ──────────────────→ 追蹤動作選擇
   │                              (selfEvolution)
   │
   ▼
自我監控追蹤 ──────────────→ 記錄互動統計
(selfMonitor)                    │
   │                             ▼
   │                      檢查停滯/突破
   │                             │
   ▼                             ▼
每 10 次互動 ──────────────→ 生成報告
   │
   ▼
每 50 決策 ───────────────→ 模式挖掘
   │
   ▼
發現失敗模式 ────────────→ 生成實驗
   │
   ▼
實驗累積數據 ────────────→ 得出結論
   │
   ▼
成功實驗 ────────────────→ 應用策略
```
